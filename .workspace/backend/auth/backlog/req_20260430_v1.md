# Backend/Auth Requirements — v1 (2026-04-30)

## Scope
Authentication flow สำหรับ yahdong — register, login, JWT access/refresh token, logout

## Requirements

### REQ-001: Register
- [ ] POST /auth/register
- [ ] Body: `{ name, email, password }`
- [ ] Validate: email format, password min 8 chars (class-validator)
- [ ] Check email ไม่ซ้ำ → 409 Conflict ถ้าซ้ำ
- [ ] Hash password ด้วย bcryptjs (saltRounds: 10)
- [ ] Return: `{ id, name, email, createdAt }` (ห้าม return password_hash)

### REQ-002: Login
- [ ] POST /auth/login
- [ ] Body: `{ email, password }`
- [ ] Verify password กับ hash
- [ ] Return: `{ accessToken, refreshToken, user: { id, name, email } }`
- [ ] accessToken: JWT, expires 15m
- [ ] refreshToken: JWT, expires 7d, เก็บ hash ใน DB (RefreshToken table)

### REQ-003: Refresh Token
- [ ] POST /auth/refresh
- [ ] Body: `{ refreshToken }`
- [ ] Verify refresh token ว่า valid + ยังไม่ revoked
- [ ] Return: `{ accessToken }` ใหม่
- [ ] 401 ถ้า token invalid/expired/revoked

### REQ-004: Logout
- [ ] DELETE /auth/logout
- [ ] Header: Authorization Bearer (access token)
- [ ] Revoke refresh token ของ user นั้นใน DB
- [ ] Return: 204 No Content

### REQ-005: JwtAuthGuard
- [ ] NestJS Guard ที่ verify Bearer token ทุก request
- [ ] Attach user payload ลงใน `req.user`
- [ ] 401 ถ้าไม่มี token หรือ invalid

### REQ-006: CurrentUser Decorator
- [ ] `@CurrentUser()` — ดึง `req.user` ออกมาใช้ใน controller
- [ ] Type: `{ id: string, email: string, name: string }`

## Notes
- ใช้ `@nestjs/passport` + `passport-jwt` strategy
- JWT_SECRET ดึงจาก `@nestjs/config` (ConfigService)
- ห้าม return `password_hash` ในทุก response
- RefreshToken table: `id, userId, tokenHash, expiresAt, revokedAt`
