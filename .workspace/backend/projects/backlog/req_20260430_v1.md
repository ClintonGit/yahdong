# Backend/Projects Requirements — v1 (2026-04-30)

## Scope
CRUD projects + member management + role-based access

## Requirements

### REQ-001: List My Projects
- [ ] GET /projects
- [ ] Requires: JwtAuthGuard
- [ ] Return projects ที่ currentUser เป็น member (ทุก role)
- [ ] Include: member count, user's own role

### REQ-002: Create Project
- [ ] POST /projects
- [ ] Body: `{ name, description?, color? }`
- [ ] สร้าง project + add creator เป็น ProjectMember role: 'owner' อัตโนมัติ
- [ ] Trigger: seed default TaskStatuses (Backlog, In Progress, Done)
- [ ] Return: created project

### REQ-003: Get Project Detail
- [ ] GET /projects/:id
- [ ] Requires: ProjectGuard (member เท่านั้น)
- [ ] Return: project + members list (id, name, avatar, role)

### REQ-004: Update Project
- [ ] PATCH /projects/:id
- [ ] Requires: ProjectGuard role >= 'owner'
- [ ] Body: `{ name?, description?, color? }`
- [ ] Return: updated project

### REQ-005: Delete Project
- [ ] DELETE /projects/:id
- [ ] Requires: ProjectGuard role = 'owner'
- [ ] Soft delete: set `deletedAt` timestamp (ไม่ลบจริง)
- [ ] Return: 204 No Content

### REQ-006: Invite Member
- [ ] POST /projects/:id/members
- [ ] Body: `{ email, role: 'member' | 'viewer' }`
- [ ] Requires: ProjectGuard role >= 'owner'
- [ ] Find user by email → 404 ถ้าไม่มี
- [ ] 409 ถ้า already member
- [ ] Return: new member object

### REQ-007: Change Member Role
- [ ] PATCH /projects/:id/members/:userId
- [ ] Body: `{ role }`
- [ ] Requires: ProjectGuard role = 'owner'
- [ ] ห้ามเปลี่ยน role ตัวเอง

### REQ-008: Remove Member
- [ ] DELETE /projects/:id/members/:userId
- [ ] Requires: ProjectGuard role = 'owner'
- [ ] ห้าม remove owner คนสุดท้าย

### REQ-009: ProjectGuard
- [ ] NestJS Guard: verify user เป็น member ของ project
- [ ] Attach `projectMember` (including role) ลงใน `req`
- [ ] Support role checking: `@Roles('owner')`, `@Roles('owner', 'member')`

## Notes
- ProjectMember roles: `owner` > `member` > `viewer`
- viewer: อ่านอย่างเดียว, ห้าม create/edit/delete tasks
