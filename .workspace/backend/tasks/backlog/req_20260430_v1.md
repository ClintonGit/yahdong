# Backend/Tasks Requirements — v1 (2026-04-30)

## Scope
CRUD tasks + kanban status management + drag & drop reorder

## Requirements

### REQ-001: List Tasks
- [ ] GET /projects/:id/tasks
- [ ] Requires: ProjectGuard
- [ ] Return: tasks grouped by statusId, include assignee (id, name, avatar)
- [ ] Optional filter: `?assigneeId=`, `?priority=`, `?sprintId=`

### REQ-002: Create Task
- [ ] POST /projects/:id/tasks
- [ ] Requires: ProjectGuard role != 'viewer'
- [ ] Body: `{ title, description?, statusId, priority?, assigneeId?, dueDate? }`
- [ ] Auto-set order: append ท้าย column นั้น
- [ ] Auto-set createdBy: currentUser.id
- [ ] Return: created task

### REQ-003: Get Task Detail
- [ ] GET /tasks/:id
- [ ] Requires: ProjectGuard (verify task belongs to project)
- [ ] Return: full task + assignee + status + comments count

### REQ-004: Update Task
- [ ] PATCH /tasks/:id
- [ ] Requires: ProjectGuard role != 'viewer'
- [ ] Body: `{ title?, description?, priority?, assigneeId?, dueDate?, statusId? }`
- [ ] Return: updated task

### REQ-005: Delete Task
- [ ] DELETE /tasks/:id
- [ ] Requires: ProjectGuard role != 'viewer'
- [ ] Hard delete
- [ ] Return: 204

### REQ-006: Move Task (drag & drop)
- [ ] PATCH /tasks/:id/move
- [ ] Body: `{ statusId, order }` — position ใหม่
- [ ] Recalculate order ของ tasks ใน column ที่ได้รับผลกระทบ
- [ ] Atomic update (transaction)
- [ ] Return: updated task

### REQ-007: List Statuses
- [ ] GET /projects/:id/statuses
- [ ] Requires: ProjectGuard
- [ ] Return: statuses ordered by `order`

### REQ-008: Create Status
- [ ] POST /projects/:id/statuses
- [ ] Requires: ProjectGuard role != 'viewer'
- [ ] Body: `{ name, color }`
- [ ] Auto-set order: append ท้าย

### REQ-009: Update Status
- [ ] PATCH /statuses/:id
- [ ] Body: `{ name?, color? }`
- [ ] Requires: ProjectGuard role != 'viewer'

### REQ-010: Delete Status
- [ ] DELETE /statuses/:id
- [ ] Requires: ProjectGuard role = 'owner'
- [ ] ห้ามลบถ้ายังมี tasks อยู่ → 409 พร้อม message ให้ย้าย tasks ออกก่อน

## Notes
- Task `order` ใช้ floating point (1000, 2000, 3000) เพื่อ reorder โดยไม่ต้อง update ทั้ง column
- Move endpoint ต้อง wrap ใน Prisma transaction
