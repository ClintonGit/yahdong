# Frontend/Project Requirements — v1 (2026-04-30)

## Scope
Project list page, create project, project settings

## Requirements

### REQ-001: Projects List Page
- [ ] Route: `/projects`
- [ ] Grid of project cards
- [ ] Card: project color bar, name, description, member count, role badge
- [ ] Loading skeleton ระหว่าง fetch
- [ ] Empty state: "ยังไม่มีโปรเจค" + ปุ่ม create

### REQ-002: Create Project Modal
- [ ] เปิดจาก "+ New Project" ใน sidebar หรือ empty state
- [ ] Fields: name (required), description (optional), color (color picker 8 preset colors)
- [ ] Validation: name required, min 2 chars
- [ ] On success: close modal + navigate to new project board
- [ ] Loading state ระหว่าง submit

### REQ-003: Project Settings Page
- [ ] Route: `/projects/:id/settings`
- [ ] Section: General — edit name, description, color
- [ ] Section: Members — list members (avatar, name, role), change role dropdown
- [ ] Section: Danger Zone — delete project (owner only, confirm dialog)

### REQ-004: Invite Member Modal
- [ ] Field: email input + role select (Member / Viewer)
- [ ] On submit: call POST /projects/:id/members
- [ ] Error: "ไม่พบ email นี้" / "เป็น member อยู่แล้ว"

### REQ-005: TanStack Query Hooks
- [ ] `useProjects()` — GET /projects
- [ ] `useProject(id)` — GET /projects/:id
- [ ] `useCreateProject()` — mutation + invalidate projects list
- [ ] `useUpdateProject(id)` — mutation
- [ ] `useDeleteProject(id)` — mutation + redirect
- [ ] `useInviteMember(projectId)` — mutation
