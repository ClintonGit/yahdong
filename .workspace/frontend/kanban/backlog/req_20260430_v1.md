# Frontend/Kanban Requirements — v1 (2026-04-30)

## Scope
Kanban board — columns, task cards, drag & drop, task detail

## Requirements

### REQ-001: KanbanBoard
- [ ] Route: `/projects/:id`
- [ ] Fetch statuses + tasks จาก API
- [ ] Render columns ตาม status order
- [ ] Horizontal scroll ถ้า columns เยอะ
- [ ] Loading skeleton

### REQ-002: KanbanColumn
- [ ] Header: status name, color dot, task count
- [ ] Scrollable task list (vertical)
- [ ] "+ Add Task" button ท้าย column
- [ ] Max height: 100vh - header

### REQ-003: TaskCard
- [ ] Title (truncate 2 lines)
- [ ] Priority badge: Urgent (red), High (orange), Medium (yellow), Low (grey)
- [ ] Assignee avatar (tooltip: name)
- [ ] Due date — แดงถ้า overdue
- [ ] Click → เปิด Task Detail Modal

### REQ-004: Drag & Drop Tasks
- [ ] @dnd-kit/sortable ใน column เดียวกัน
- [ ] @dnd-kit/core DragOverlay ระหว่าง drag
- [ ] Drop ข้าม column ได้
- [ ] Optimistic update: ย้าย task ใน UI ทันที ก่อน API call
- [ ] Rollback ถ้า API error

### REQ-005: Drag & Drop Columns
- [ ] @dnd-kit/sortable สำหรับ column reorder
- [ ] Owner/member เท่านั้น (viewer drag ไม่ได้)

### REQ-006: Task Detail Modal
- [ ] Inline edit: title, description (textarea)
- [ ] Priority select
- [ ] Assignee select (project members)
- [ ] Due date picker
- [ ] Status select
- [ ] Delete button (ยืนยันก่อนลบ)

### REQ-007: Create Task Inline
- [ ] Click "+ Add Task" → show input ใน column
- [ ] Enter หรือ click "Add" → create task
- [ ] Escape → cancel
- [ ] Auto-focus เมื่อเปิด

### REQ-008: Optimistic Update
- [ ] useMoveTask: update local state ก่อน mutate API
- [ ] ถ้า API error → revert + show toast error

## Notes
- DndContext ครอบ KanbanBoard ทั้งหมด
- Sensors: PointerSensor + KeyboardSensor (accessibility)
- Column drag handle: handle icon ด้านซ้ายของ column header
