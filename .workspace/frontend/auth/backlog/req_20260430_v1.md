# Frontend/Auth Requirements — v1 (2026-04-30)

## Scope
Login/Register pages + token management + protected routes

## Requirements

### REQ-001: Register Page
- [ ] Route: `/register`
- [ ] Fields: name, email, password, confirm password
- [ ] Validation (zod): email format, password min 8 chars, passwords match
- [ ] Error display inline ใต้ field
- [ ] On success → redirect to `/` (projects page)
- [ ] Loading state ระหว่าง API call

### REQ-002: Login Page
- [ ] Route: `/login`
- [ ] Fields: email, password
- [ ] Validation (zod): email format, password required
- [ ] On success → redirect to `/`
- [ ] Error: "อีเมลหรือรหัสผ่านไม่ถูกต้อง" (ห้ามบอกว่าอันไหนผิด)

### REQ-003: Auth Store (Zustand)
- [ ] State: `user`, `accessToken`, `refreshToken`, `isAuthenticated`
- [ ] Actions: `setAuth(user, tokens)`, `clearAuth()`
- [ ] Persist: accessToken + refreshToken ใน localStorage
- [ ] Rehydrate: load tokens on app start

### REQ-004: Axios Interceptor
- [ ] Request interceptor: attach `Authorization: Bearer {accessToken}`
- [ ] Response interceptor: ถ้า 401 → call POST /auth/refresh อัตโนมัติ
- [ ] ถ้า refresh ไม่ได้ → clearAuth() + redirect `/login`

### REQ-005: ProtectedRoute
- [ ] HOC/component wrap routes ที่ต้อง login
- [ ] ถ้า `!isAuthenticated` → redirect `/login`
- [ ] แสดง loading spinner ระหว่างตรวจสอบ

### REQ-006: Logout
- [ ] Call DELETE /auth/logout
- [ ] clearAuth() ใน store
- [ ] Redirect → `/login`

## Notes
- ห้าม store password ใน state
- accessToken ไม่ควรเก็บใน localStorage นาน — เก็บใน memory หรือ sessionStorage แต่ refreshToken เก็บ localStorage ได้
