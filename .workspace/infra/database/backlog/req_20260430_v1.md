# Infra/Database Requirements — v1 (2026-04-30)

## Scope
Prisma schema + migrations + seed สำหรับ Phase 1

## Requirements

### REQ-001: Prisma Schema
- [ ] Model: `User` — id, name, email (unique), passwordHash, avatar, createdAt
- [ ] Model: `RefreshToken` — id, userId, tokenHash, expiresAt, revokedAt
- [ ] Model: `Organization` — id, name, slug (unique), createdAt
- [ ] Model: `OrgMember` — orgId, userId, role (owner/admin/member)
- [ ] Model: `Project` — id, orgId, name, slug, description, color, deletedAt, createdAt
- [ ] Model: `ProjectMember` — projectId, userId, role (owner/member/viewer)
- [ ] Model: `TaskStatus` — id, projectId, name, color, order
- [ ] Model: `Task` — id, projectId, title, description, statusId, priority, assigneeId, sprintId, dueDate, order, createdBy, createdAt
- [ ] Model: `Comment` — id, taskId, userId, body, createdAt
- [ ] Enums: `OrgRole`, `ProjectRole`, `TaskPriority`

### REQ-002: Initial Migration
- [ ] `npx prisma migrate dev --name init`
- [ ] ตรวจว่า migration ผ่านบน local PostgreSQL

### REQ-003: Auto Migration on Startup
- [ ] `main.ts` รัน `prisma.$executeRaw` หรือ `prisma migrate deploy` ก่อน `app.listen()`
- [ ] ใช้ `prisma migrate deploy` (production-safe, ไม่สร้าง migration ใหม่)

### REQ-004: Prisma Seed
- [ ] `prisma/seed.ts`
- [ ] Seed: 1 user (admin@yahdong.test / password: test1234)
- [ ] Seed: 1 project "Demo Project" + default statuses
- [ ] Script: `npx prisma db seed`

## Notes
- ใช้ `cuid()` เป็น default id (Prisma built-in)
- Task order: Float เพื่อรองรับ reorder โดยไม่ต้อง update ทุก row
- deletedAt บน Project เพื่อ soft delete
